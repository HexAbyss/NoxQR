# NOX Testing Guide

## Testing Layers

NOX currently uses three complementary validation layers locally.

### 1. Backend Unit Tests

Command:

```bash
bash scripts/run-backend-unit-tests.sh
```

What it does:

- starts a disposable Rust container
- mounts `backend/`
- runs `cargo test`

What it protects:

- matrix role classification
- importance scoring behavior
- renderer contract behavior in the backend
- request validation logic
- decoder-backed scanability for representative styles
- reliability scoring and safety-bias heuristics

### 2. Frontend Typecheck

Command:

```bash
bash scripts/run-frontend-typecheck.sh
```

What it does:

- starts a disposable Node container
- installs frontend dependencies without writing a lockfile
- runs `npm run typecheck`

What it protects:

- API payload and response typing
- store and component contract stability
- UI exposure of renderer styles
- reliability telemetry mapping in the preview layer

### 3. Full-Stack Regression

Command:

```bash
bash scripts/run-regression-tests.sh
```

What it does:

- builds and starts frontend and backend through Docker Compose
- waits for both services to become ready
- launches Playwright in a disposable container
- runs `tests/e2e/regression.spec.mjs`
- shuts the Compose stack down automatically when the run ends

Useful environment variables:

- `NOX_E2E_FRONTEND_URL`
- `NOX_E2E_BACKEND_URL`
- `NOX_E2E_KEEP_STACK=1` to keep containers alive for debugging after the run

What it protects:

- `GET /health`
- `POST /generate`
- SVG, PNG, and validation contract stability across all supported styles
- reliability telemetry visibility in the preview
- export button behavior in light and dark themes
- mobile containment and overflow regressions

## Complete Battery

Command:

```bash
bash scripts/run-test-battery.sh
```

This is the recommended local verification command before pushing changes.

It runs, in order:

1. backend unit tests
2. frontend typecheck
3. full-stack regression

## CI Coverage

GitHub Actions is configured in `.github/workflows/regression.yml`.

Current CI jobs:

- backend unit tests
- full-stack regression

The local battery is broader than CI because it also includes the frontend typecheck.

## Test Files and Scripts

### Scripts

- `scripts/run-backend-unit-tests.sh`: Rust unit tests inside Docker
- `scripts/run-frontend-typecheck.sh`: frontend TypeScript validation inside Docker
- `scripts/run-regression-tests.sh`: Compose-backed Playwright regression run
- `scripts/run-test-battery.sh`: local one-command verification path
- `scripts/capture-readme-screenshots.sh`: regenerates docs imagery from the running product
- `scripts/capture-readme-screenshots.mjs`: Playwright capture logic for docs assets

### Test Sources

- `tests/e2e/regression.spec.mjs`: end-to-end regression suite

## Generated Artifacts

### `test-results/`

`test-results/` is generated by Playwright runs.

It contains execution artifacts such as:

- `.last-run.json`
- run summaries or future screenshots/traces if enabled

This directory is ignored by git and should not be treated as source.

## Practical Recommendation

Use these commands by intent:

- backend-only change: `bash scripts/run-backend-unit-tests.sh`
- frontend-only change: `bash scripts/run-frontend-typecheck.sh`
- behavior change or release candidate: `bash scripts/run-test-battery.sh`
